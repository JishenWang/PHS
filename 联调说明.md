# 提示词：给团队成员一份“下拉即用”的数据库-后端-前端联调标准手册，确保不同机器可以快速连通。

## 1. 数据库统一（使用同一份 SQL）

项目统一使用：`c:\Users\15693\Desktop\pet_hospital.sql`

> 强制要求：数据库字段名必须与后端 Mapper SQL 中的列名完全一致，不允许“猜字段”。

### 1.1 建议导入命令（MySQL 8）

```bash
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS pet_hospital DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;"
mysql -u root -p pet_hospital < c:\Users\15693\Desktop\pet_hospital.sql
```

### 1.2 导入后快速核对

```sql
SELECT COUNT(1) AS user_count FROM sys_user;
SELECT COUNT(1) AS pet_count FROM pet;
SELECT COUNT(1) AS register_count FROM register_record;
SELECT COUNT(1) AS order_count FROM order_info;
```

## 2. 后端统一配置（Spring Boot）

后端配置文件：`PHS/pet-hospital/src/main/resources/application.yml`

当前已支持通过环境变量覆盖数据库/Redis配置，团队成员只需设置自己的本地环境变量：

- `DB_HOST`（默认 `localhost`）
- `DB_PORT`（默认 `3306`）
- `DB_NAME`（默认 `pet_hospital`）
- `DB_USERNAME`（默认 `root`）
- `DB_PASSWORD`（默认 `root`）
- `REDIS_HOST`（默认 `localhost`）
- `REDIS_PORT`（默认 `6379`）
- `REDIS_DB`（默认 `0`）

### 2.1 推荐本地环境变量示例（Windows PowerShell）

```powershell
$env:DB_HOST="localhost"
$env:DB_PORT="3306"
$env:DB_NAME="pet_hospital"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="root"
$env:REDIS_HOST="localhost"
$env:REDIS_PORT="6379"
$env:REDIS_DB="0"
```

### 2.2 后端启动命令

```bash
cd PHS/pet-hospital
mvn spring-boot:run
```

## 3. 前端统一配置（Vue3 + Vite）

前端会读取以下变量：

- `VITE_BASE_API`：请求前缀（开发环境建议 `/api`）
- `VITE_API_TARGET`：Vite 代理目标（例如 `http://localhost:8080`）

### 3.1 建议开发环境 `.env.development`

```env
VITE_BASE_API=/api
VITE_API_TARGET=http://localhost:8080
VITE_APP_TITLE=宠物医院管理系统
```

### 3.2 前端启动命令

```bash
cd PHS/pet-hospital-frontend
npm install
npm run dev
```

---

## 4. 联调启动顺序（必须一致）

1. 启动 MySQL 并确认 `pet_hospital` 库已导入。
2. 启动后端 `pet-hospital`（默认 `8080`，上下文 `/api`）。
3. 启动前端 `pet-hospital-frontend`（默认 `3000`）。
4. 浏览器访问前端后，前端请求会通过 `/api` 代理到后端。

---

## 5. 联通性检查接口

可先访问这两个接口验证“前端 -> 后端 -> 数据库”基础链路：

- `GET http://localhost:8080/api/desk/ping`
- `GET http://localhost:8080/api/desk/debug/table-stats`

如果 `table-stats` 中核心表（`sys_user`、`pet`、`register_record`、`order_info`）都能返回计数，说明数据库联通正常。

## 7. 常见问题与处理

### 7.1 前端能打开但接口 404

- 检查前端 `VITE_BASE_API` 是否为 `/api`
- 检查 `VITE_API_TARGET` 是否指向后端端口（默认 `http://localhost:8080`）
- 检查后端是否已成功启动且上下文包含 `/api`

### 7.2 后端启动失败（数据库连接）

- 检查本机 MySQL 是否启动
- 检查数据库名是否为 `pet_hospital`
- 检查环境变量中的 `DB_USERNAME`、`DB_PASSWORD`

### 7.3 SQL 运行报字段不存在

- 先确认使用的是统一 SQL：`c:\Users\15693\Desktop\pet_hospital.sql`
- 不要恢复旧兼容 Mapper（`OrderMapper/RegisterMapper/ReserveMapper/ServiceItemMapper`）
- 以 `PHS/pet-hospital/src/main/resources/字段一致性清单.md` 为唯一映射校验基线
